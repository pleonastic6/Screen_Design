# Information Architecture

## Primäre Navigationsstruktur

1. **Karte**
   - Scooter in der Nähe
   - Ladehubs
   - Statusfilter
2. **Fahrt**
   - aktiver Ride-Status
   - Pause / Weiterfahrt
   - Rückgabe
3. **Buchungen**
   - Reservierung aktiv
   - vergangene Fahrten
   - Kostenübersicht
4. **Profil**
   - Kontodaten
   - Zahlungsmethode
   - Hilfe / Regeln

## Zentrale Screens

### 1. Onboarding / Registrierung
- Konto erstellen
- Mobilitätsregeln grob erklären

### 2. Home / Karte
- Default-Screen
- Scooter-Marker mit Status
- Ladehubs als feste Points of Interest

### 3. Scooter-Detail
- Name / ID
- Akkustand
- Distanz
- reservierbar / direkt verfügbar
- CTA: reservieren oder entsperren

### 4. Reservierungsstatus
- verbleibende Zeit
- Standort des Scooters
- CTA: Navigation / Reservierung aufheben

### 5. Unlock-Screen
- Zuordnung App ↔ Scooter
- Bestätigung der richtigen Fahrzeug-ID
- Remote-Unlock

### 6. Fahrstatus
- Dauer
- aktuelle Kosten
- Akku
- Aktionen: pausieren, zurückgeben

### 7. Parkmodus
- Hinweis, dass Abrechnung weiterläuft
- CTA: weiterfahren / endgültig zurückgeben

### 8. Rückgabe-Check
- Standort erlaubt?
- Bonus bei Ladehub-Rückgabe
- Abschluss-CTA

### 9. Ride Summary
- Dauer
- Preis
- Freiminuten-Bonus
- Abschlussbestätigung

## Design-Prinzipien

- wenige Hauptbereiche
- klare Statuswechsel
- keine versteckten Regeln
- Kartenfokus als Primärnutzung
